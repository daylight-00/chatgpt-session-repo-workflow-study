# Compact evidence

This directory contains small raw or near-raw evidence suitable for Git history.

It is intentionally different from Release assets:

- `evidence/` contains compact probes, selected raw logs, checksums, and reconstructible encodings;
- `release-assets/` tracks complete large bundles and their verification state.

A file in `evidence/` should have a clear claim-to-evidence role and should not be a transient cache.
